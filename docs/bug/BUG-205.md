## 描述

SSE `onMessage` 回调使用 `||` 条件判断完成状态，且无幂等性保护，`handleSaveResults` 可能被多次调用。

## 根因

第 644 行：
```javascript
if (data.status === 'completed' || data.progress >= 100) {
    handleSaveResults()
}
```

- 使用 `||` 意味着 status=completed **或** progress>=100 都会触发保存
- 如果后端发送两条 SSE 消息（先发 progress=100，再发 status=completed），`handleSaveResults` 被调用两次
- `handleSaveResults`（第 669-679 行）无幂等性检查

## 影响

- 两次调用 `saveImportRequests()` 的 `POST /save/` 请求
- 理论上可能导致重复创建 ApiRequest 记录（取决于 `select_for_update` 和事务隔离级别）
- Step 5（currentStep=5）可能被错误设置

## 修复建议

在 `handleSaveResults` 开头添加幂等性守卫：

```javascript
function handleSaveResults() {
    if (generateProgress.value >= 100) return
    generateProgress.value = 100
    // ... rest of the function
}
```
