## 描述

SSE 的 `onprogress` 回调每次都完整分割 `xhr.responseText` 并处理所有行，导致每条历史消息都被重复处理。

## 根因

```javascript
xhr.onprogress = () => {
    const lines = xhr.responseText.split('\n')
    for (const line of lines) {
        if (line.startsWith('data: ')) {
            onMessage(JSON.parse(line.slice(6)))
        }
    }
}
```

每次 `onprogress` 触发时重新处理 `responseText` 的 **全部内容**，而非仅处理新到达的数据。如果后端发送了 N 条消息，第 N 次回调会处理全部 N 条消息。

## 影响

- 叠加 BUG-205 的 `handleSaveResults` 无幂等问题，显著增加重复保存概率
- 重复的 `generateProgress` / `generateMessage` 状态更新导致不必要的组件重渲染
- 理论上可能在 partial line 边界处出现 JSON 解析错误（虽然后续 onprogress 会修复）

## 修复建议

维护 `lastIndex` 指针，只处理上次处理后的新数据：

```javascript
let lastIndex = 0
xhr.onprogress = () => {
    const newData = xhr.responseText.slice(lastIndex)
    lastIndex = xhr.responseText.length
    // 只处理 newData 中的行
}
```
