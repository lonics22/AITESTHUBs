## 描述

SSE 解析器只处理 `data:` 前缀的行，完全忽略 `event:` 字段。如果后端将来发送不同事件类型（如 `event: progress`、`event: complete`、`event: error`），所有事件都会通过同一个 `onMessage` 回调处理，无法区分。

## 根因

```javascript
for (const line of lines) {
    if (line.startsWith('data: ')) {
        onMessage(JSON.parse(line.slice(6)))
    }
}
```

`event:` 行被完全忽略，仅检查 `data:` 行。

## 影响

- 后端无法按事件类型分发不同的处理逻辑
- 错误事件无法被特殊处理（如显示单独的错误提示）
- 当前状态虽未暴露问题（后端只发一种事件），但限制了未来的扩展

## 修复建议

在解析器中追踪当前事件类型：

```javascript
let currentEvent = 'message'
for (const line of lines) {
    if (line.startsWith('event: ')) {
        currentEvent = line.slice(7).trim()
    } else if (line.startsWith('data: ')) {
        onMessage(JSON.parse(line.slice(6)), currentEvent)
        currentEvent = 'message'
    }
}
```
