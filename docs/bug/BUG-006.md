## 描述

步骤 6 保存完成后，成功提示始终显示"已成功创建 0 个 API 请求"，尽管数据库中实际已创建请求。此问题同时影响预览路径（步骤 5 → 直接保存）和 SSE 回调保存路径。

## 根因

前端读取响应字段名与后端返回不匹配：

前端：
```js
// 保存回调
savedCount.value = res.data?.count || res.data?.created_count || 0

// 预览直接保存
savedCount.value = previewRes.data?.count || previewRes.data?.requests?.length || 0
```

后端 `/save/` 返回：
```json
{"message": "...", "collections_created": [1], "requests_created": [101, 102, 103]}
```

后端 `/preview/` 返回原始列表（非含 `.requests` 的对象）。

## 复现步骤

1. 完成步骤 5 生成
2. 点击保存或预览后确认保存
3. 成功消息显示"已成功创建 0 个 API 请求"
4. 实际检查数据库 → 请求已成功创建

## 预期行为

正确显示实际创建的请求数量，如"已成功创建 3 个 API 请求"。

## 修复建议

```js
savedCount.value = res.data?.requests_created?.length || 0
```
