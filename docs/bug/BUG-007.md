## 描述

步骤 3 中 `field_type='multi_param'` 类型的参数表格，端点路径列（第二列）始终为空。后端发送的字段名是 `endpoint`（如 `"GET /users"`），前端读取 `row.path` 不存在。

## 根因

前端代码：
```html
<span class="endpoint-method">{{ row.method }}</span>
<span class="endpoint-path">{{ row.path }}</span>
```

后端发送的 multi_param 选项：
```python
{"param_name": "search", "method": "GET", "endpoint": "GET /api/users", ...}
```

无 `path` 字段。

## 复现步骤

1. 上传含 Query/Header 参数的文档
2. 步骤 3 查看多参数表格
3. 端点路径列（第二列）始终为空
4. 用户无法判断参数属于哪个端点

## 预期行为

表格中正确显示 HTTP 方法和端点路径。

## 修复建议

```html
<span class="endpoint-method">{{ row.method }}</span>
<span class="endpoint-path">{{ row.endpoint }}</span>
```
