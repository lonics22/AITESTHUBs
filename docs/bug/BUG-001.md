## 描述

步骤 3「AI 问题列表」中，`field_type='string'` 和 `field_type='select'` 类型问题的输入框/下拉框无法收集用户输入。前端模板使用 `question.param_name` 作为 v-model 绑定 key，但后端返回的 AIQuestion 数据结构中不存在 `param_name` 字段（后端用 `id` 作为唯一标识）。导致：

1. 问题标题渲染为 `undefined`（模板读 `question.question || question.param_name`）
2. `answers[question.param_name]` → `answers[undefined]`，用户输入被丢弃
3. string/select 问题的回答无法传递到生成阶段

## 涉及范围

- 步骤 3 的 `string` 类型文本输入框全部失效
- 步骤 3 的 `select` 类型下拉选择框全部失效
- 生成的 API 请求缺少用户应填写的参数值

## 根因

前后端数据契约不一致。后端 `AIQuestion` 字段：
```
{ id, category, title, description, field_type, options, ... }
```

前端模板读取：
```
question.param_name  ← 不存在
question.question    ← 不存在（后端字段是 title）
```

## 复现步骤

1. 上传一个带认证头或多参数的文档
2. 步骤 3 加载问题列表
3. 观察问题标题显示为 `undefined`
4. 在文本输入框输入值、在下拉框选择选项
5. 点击下一步提交
6. 查看提交的 JSON → string/select 问题的值为空

## 预期行为

- 问题标题应显示后端返回的 `title` 字段
- 用户输入应绑定到 `answers[question.id]` 或类似存在的 key
- 提交 answers API 时应包含所有 string/select 问题的值

## 修复建议

前端 `AIImportWizard.vue` 中：
- `question.question` → `question.title`
- `answers[question.param_name]` → `answers[question.id]`
