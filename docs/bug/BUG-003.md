## 描述

`env_var_mapping` 类型问题中，前端构建的 `environmentVars` 数据格式为**对象数组**：
```json
[{"original_value": "api.example.com", "var_name": "BASE_URL"}]
```

但后端的 `generate_requests` → `_replace_env_vars` 中以**字典**方式遍历：
```python
for orig, var_key in environment_vars.items():
```

对数组调用 `.items()` 抛出 `AttributeError`，answers API 返回 500，任务标记为 failed。

## 涉及范围

- 任何含环境变量配置的导入任务都会失败
- URL 模板 `{{var_name}}` 替换完全不可用

## 根因

前端 `AIImportWizard.vue` 第 611-615 行：
```js
environmentVars.push(...(q.variables || []).map(v => ({
  original_value: v.original_value,
  var_name: v.var_name
})));
```

后端 `AIImportTask.environment_vars` 字段 help_text 指明格式为 `{"original_value": "{{var_name}}"}`。

## 复现步骤

1. 上传文档，进入步骤 3
2. 在环境变量映射部分添加一条映射（如 `api.example.com` → `BASE_URL`）
3. 点击生成（提交 answers）
4. /answers/ API 返回 500 错误
5. 任务状态变为 failed

## 预期行为

环境变量映射正确提交，后端能按字典格式解析，URL/参数中正确替换为 `{{BASE_URL}}`。

## 修复建议

**Option A（改前端）**：提交前将数组转为字典格式：
```js
const envVarDict = {};
environmentVars.forEach(v => { envVarDict[v.original_value] = v.var_name; });
```

**Option B（改后端）**：`_replace_env_vars` 同时兼容数组和字典格式。
