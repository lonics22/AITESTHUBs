## 描述

Upload action 的进度值顺序为：10（上传中）→ 20（解析中）→ 50（AI 分析中）→ 70（分类完成）→ **40**（回到 waiting_user 状态）。进度条从 70 回退到 40，造成视觉跳跃。

## 根因

第 2933 行设置 `progress = 70`，第 2940 行设置 `progress = 40`：

```python
# Phase 1 完成 → progress = 70
task.ai_classification = classification
task.progress = 70
task.save(update_fields=['ai_classification', 'progress'])

# Phase 2 完成 → 状态变为 waiting_user，但 progress 被改回 40
task.ai_questions = questions
task.status = 'waiting_user'
task.progress = 40
task.save(update_fields=['ai_questions', 'status', 'progress'])
```

`progress = 40` 是 upload 完成的最终值（代表"等待用户配置"），但中间阶段 progress 已经到 70，最终值比中间值低。

## 影响

- Step 2（AI 分析）的进度条从 70% 回跳到 40%
- 用户感知上像是"进度倒退"，不符合用户预期

## 修复建议

最终等待用户的进度值应 >= 所有中间进度值。可以将最终进度调整为 70（与分类完成一致），或在完成状态只保留最终进度 70，取消中间的 40 值。
