## 描述

`answers` 第 3026 行设置 `generated_summary = {'requests': generated_requests}`，未包含 `request_count` 字段。预览和保存依赖于 `len(requests)` 获取计数，但前端或 SSRE 进度可能期望 `request_count` 字段。

## 根因

```python
task.generated_summary = {'requests': generated_requests}
# 缺少 request_count: len(generated_requests)
```

而后在 save action（第 3168-3171 行）添加了 `collections_created` 和 `requests_created`，如果前端或中间件依赖 `request_count`，将得到 `None`。

```python
task.generated_summary['collections_created'] = created_collections
task.generated_summary['requests_created'] = created_request_ids
```

但 `request_count` 始终未设置。

## 影响

- 任何依赖 `generated_summary.request_count` 的代码将得到 `None`
- 前端进度显示可能不准确

## 修复建议

在设置 `generated_summary` 时同时添加 `request_count`：

```python
task.generated_summary = {
    'requests': generated_requests,
    'request_count': len(generated_requests),
}
```
