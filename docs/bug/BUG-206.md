# BUG-206：Fallback setTimeout 未在卸载/重置时清理

---

| 字段 | 值 |

| **发现日期** | 2026-06-30 |
| **模块** | 前端 - AIImportWizard.vue SSE fallback |
| **优先级** | P2 - 组件卸载后操作 |
| **状态** | 待修复 |
| **修复人** | — |
| **验证人** | — |

---

## 描述

SSE 订阅后的 30 秒 fallback `setTimeout` 未保存 timer ID，无法在 `onUnmounted` 或 `resetWizard` 中清理。

## 根因

第 655-666 行：
```javascript
setTimeout(async () => {
    if (currentStep.value === 4 && generateProgress.value < 100) {
        // ... fallback save
    }
}, 30000)
```

未保存返回值：`const fallbackTimer = setTimeout(...)`，无法 `clearTimeout(fallbackTimer)`。

## 影响

- 用户快速点击"继续导入"重置向导后，旧 timer 会在新任务上触发 `saveImportRequests`
- 组件卸载后 timer 依然执行，操作解绑的响应式状态，可能导致控制台错误或异常行为
- resetWizard 中置空的 `taskId` 被 timer 引用

## 修复建议

保存 timer ID，在 `onUnmounted` 和 `resetWizard` 中调用 `clearTimeout()`

```javascript
const fallbackTimer = ref(null)
// 保存：fallbackTimer.value = setTimeout(...)
// 清理：clearTimeout(fallbackTimer.value)
```
