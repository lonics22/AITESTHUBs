## 描述

步骤 3 中 `field_type='env_var_mapping'` 类型的问题页面，环境变量映射表格始终为空。前端渲染时读取 `question.variables`，但后端将检测到的环境变量数据放在 `question.options` 中。

## 根因

前端：
```html
<div v-for="(envVar, evIndex) in question.variables || []" ...>
```

后端 `generate_questions()` 生成的 env_var 问题结构：
```python
AIQuestion(
    category="env_var",
    options=[  # 后端放在 options 里
        {"original_value": "api.example.com", "endpoints": [...]},
        ...
    ],
    ...
)
```

前端读 `question.variables` → undefined → 空列表。

## 复现步骤

1. 上传含多个域名或常见占位符的文档
2. 步骤 3 查看环境变量映射表格
3. 表格为空，AI 检测到的变量未自动填充
4. 用户必须手动点击"添加变量"逐条添加

## 预期行为

AI 检测到的环境变量应自动填充到映射表的"原始值"列，用户只需填写变量名。
