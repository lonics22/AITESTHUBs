# BUG-208：handleGenerate 无问题自动跳转时 Step 卡住

## 描述

当文档无 manual 参数（questions 为空）时，`handleAutoAnalysis` 先将 `currentStep` 设为 4，再调用 `await handleGenerate()`。如果 `handleGenerate` 抛出异常，用户卡在 Step 4 且无导航按钮。

## 根因

```javascript
// handleAutoAnalysis:
} else if (success) {
    currentStep.value = 4          // 先切换步骤
    await handleGenerate()          // 再调用生成，可能抛异常
}
```

Step 4 的导航按钮：
- "上一步"：`v-if="currentStep > 0 && currentStep < 5"` — **会渲染**（4 < 5）
- "下一步"：`v-if="currentStep < 4"` — **不会渲染**
- 但 `handleGenerate` 失败后 `stepLoading` 可能变为 false，用户可点"上一步"回到 Step 3？实际上如果 questions 为空，没有 Step 3，用户不知道回退到哪里。

## 影响

- 极低概率：仅在无 manual 参数且 handleGenerate 同时失败的场景触发
- 但一旦触发，用户只能刷新页面重新开始

## 修复建议

在 `handleGenerate` 前不移步，或使用 try/catch 兜底：

```javascript
} else if (success) {
    try {
        currentStep.value = 4
        await handleGenerate()
    } catch (e) {
        currentStep.value = 3  // 回退到问题列表步骤
    }
}
```
