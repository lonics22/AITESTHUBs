## 描述

`POST /upload/` 端点使用 `file.read().decode('utf-8')` 将整个上传文件读入内存，但未对文件大小做任何校验。恶意或误操作上传 GB 级文件可直接导致服务器内存耗尽（OOM）。

## 根因

views.py 中 upload action 缺少文件大小检查：
```python
content = file.read().decode('utf-8')  # 无 size 校验
```

## 复现步骤

1. 构造一个 1GB 的 JSON 文件
2. POST 到 `/api-testing/ai-import/upload/`
3. 服务器内存使用飙升，可能触发 OOM Killer

## 预期行为

上传前校验 `file.size`，超过限制（建议 50MB）返回 413 Payload Too Large：
```python
MAX_FILE_SIZE = 50 * 1024 * 1024  # 50MB
if file.size > MAX_FILE_SIZE:
    return Response({'error': f'文件大小不能超过 50MB'}, status=413)
```
