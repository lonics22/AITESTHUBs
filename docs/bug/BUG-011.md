## 描述

`/answers/` 和 `/save/` 端点使用 `select_for_update()` 行级锁以控制并发安全，但未包裹在 `transaction.atomic()` 上下文管理器中。在 Django 中，无活跃数据库事务时 `select_for_update()` 的行锁在查询完成后立即释放，无法起到互斥作用。

## 根因

```python
# 缺少 @transaction.atomic 装饰器
task = AIImportTask.objects.select_for_update().get(id=task_id)
# 锁在此处立即释放（无事务）
```

## 复现步骤（高并发场景）

1. 两个并发请求同时到达 `/answers/`（同一 task_id）
2. 两个请求都读取到 `status='waiting_user'`
3. 两个请求都执行生成逻辑
4. 生成结果相互覆盖或重复

## 预期行为

```python
from django.db import transaction

@transaction.atomic
def answers(self, request, id):
    task = AIImportTask.objects.select_for_update().get(id=id)
    # ... 锁在整个事务期间保持
```
