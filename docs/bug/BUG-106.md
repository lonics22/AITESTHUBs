## 描述

`preprocess_images()` 方法使用 `asyncio.get_event_loop().run_in_executor(ThreadPoolExecutor)` 在线程池中执行 `process_single_image`，而该函数内部又调用 `asyncio.run()` 创建新的事件循环来执行 `call_openai_compatible_api`。

层级关系：
```
asyncio.gather()
  → loop.run_in_executor(thread_pool)  # 同步闭包
    → asyncio.run(coro)                 # 创建新事件循环
      → call_openai_compatible_api()    # httpx.AsyncClient
```

每个工作线程创建并销毁一个事件循环，开销较大。

## 影响

- 处理大量图片（10+）时性能不佳
- 代码理解难度高
- 复杂度与收益不匹配

## 修复建议

将 `process_single_image` 改为 async 函数，直接 `asyncio.gather()` 协程而不是通过线程池：
```python
async def process_single_image(match):
    ...

tasks = [process_single_image(m) for m in matches]
results = await asyncio.gather(*tasks, return_exceptions=True)
```
