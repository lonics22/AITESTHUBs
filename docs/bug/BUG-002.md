## 描述

步骤 3 中 `field_type='select'` 类型问题的下拉选项显示为 `[object Object]`。后端发送的选项格式为 `{label, value}` 对象数组，但前端 `el-option` 直接把整个对象作为 label 和 value 渲染。

## 根因

前端代码：
```html
<el-option v-for="opt in question.options" :key="opt" :label="opt" :value="opt" />
```

应改为：
```html
<el-option v-for="opt in question.options" :key="opt.value" :label="opt.label" :value="opt.value" />
```

## 复现步骤

1. 上传一个带认证信息的文档（如 Postman 集合含 Authorization header）
2. 步骤 3 查看认证方式选择器
3. 下拉选项全部显示为 `[object Object]`
4. 无法选择正确的认证方式

## 预期行为

下拉框显示"Bearer Token"、"Basic Auth"、"API Key"等可读文字。
