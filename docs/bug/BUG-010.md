## 描述

前端文件选择器配置 `accept=".json,.yaml,.yml"` 允许上传 YAML 文件，但后端使用 `json.loads()` 解析，未调用 `PyYAML`（虽然在 requirements.txt 中已列出）。YAML 格式的 Swagger/OpenAPI 文件上传后收到误导性错误"文件不是有效的 JSON"。

## 根因

views.py upload action：
```python
content = file.read().decode('utf-8')
try:
    data = json.loads(content)
except json.JSONDecodeError:
    return Response({'error': '文件不是有效的 JSON'}, ...)
```

未尝试 YAML 解析。

## 复现步骤

1. 准备一个 `.yaml` 格式的 OpenAPI 3.0 规格文件
2. 前端选择文件上传
3. 返回错误"文件不是有效的 JSON"
4. 用户需手动将 YAML 转为 JSON 再上传

## 预期行为

后端应检测文件扩展名或尝试 YAML 解析：
```python
import yaml

if file.name.endswith(('.yaml', '.yml')):
    data = yaml.safe_load(content)
else:
    data = json.loads(content)
```

或统一先尝试 JSON，失败后尝试 YAML。
