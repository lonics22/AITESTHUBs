## 描述

Upload 解析器根据文件扩展名（`.json` / `.yaml` / `.yml`）选择解析策略（`json.loads` / `yaml.safe_load`），而非根据文件实际内容。与设计文档 TC 1.8 要求"以文件实际内容为准，不依赖后缀判断"相悖。

## 根因

```python
_, ext = os.path.splitext(file.name.lower())
if ext == '.json':
    content = json.loads(raw)        # YAML 内容会解析失败
elif ext in ('.yaml', '.yml'):
    content = yaml.safe_load(raw)    # JSON 内容也能被 YAML 解析
```

- `.json` 扩展但内容是 YAML → `json.loads` 抛出异常 → 返回 400 错误
- `.yaml` 扩展但内容是 JSON → `yaml.safe_load` 可能成功（YAML 是 JSON 超集）

## 复现步骤

1. 准备包含 YAML 内容的 API 文档
2. 将扩展名改为 `.json`
3. 上传 → 返回 400 错误"文件不是有效的 JSON"
4. 实际上文件内容是有效的 YAML 格式

## 影响

- 用户如果因为系统限制只允许 .json 扩展名上传，将无法使用 YAML 格式文档
- 不符合设计规范 TC 1.8
- 用户体验不佳：按系统提示换文件名后反而报错

## 修复建议

优先根据文件扩展名尝试解析，若失败则自动尝试另一种格式：

```python
_, ext = os.path.splitext(file.name.lower())
if ext == '.json':
    try:
        content = json.loads(raw)
    except json.JSONDecodeError:
        # JSON 解析失败，尝试 YAML
        content = yaml.safe_load(raw)
elif ext in ('.yaml', '.yml'):
    try:
        content = yaml.safe_load(raw)
    except Exception:
        # YAML 解析失败，尝试 JSON
        content = json.loads(raw)
```
