## 描述

`formatMarkdown()` 方法中 HTML 实体转义（`&` → `&amp;`）在图片正则替换之前执行。如果图片 URL 包含查询参数（如 `https://example.com/image.png?token=abc&expires=123`），URL 中的 `&` 被转义为 `&amp;`，最终 `<img>` 标签的 `src` 属性损坏。

## 根因

处理顺序：
1. `html.replace(/&/g, '&amp;')` — 全局将所有 `&` 转义
2. `html.replace(/!\[(.*?)\]\((.*?)\)/g, '<img src="$2"...>')` — 此时 URL 中的 `&` 已变为 `&amp;`

## 复现步骤

1. 粘贴一张图片，其上传 URL 包含 `&` 参数
2. 点击「生成测试用例」进入预览
3. 图片无法加载（src 属性值为 ...?token=abc&amp;expires=123）

## 预期行为

调整处理顺序：先处理图片替换，再处理 HTML 转义。或图片替换中使用原始未转义的 URL。
