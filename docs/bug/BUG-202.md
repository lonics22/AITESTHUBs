## 描述

`generate_requests()` 函数解析端点的 `path` 字段生成 URL，但从未将用户回答的 Base URL（来自 `url_domain` 问题的答案）拼接到路径前。所有生成的请求 URL 都是相对路径。

## 根因

`generate_requests()` 第 721 行：

```python
url = re.sub(r"\{(\w+)\}", r"{{\1}}", ep.get("path", ""))
```

仅做了路径参数模板替换（`{petId}` → `{{petId}}`），但没有检查 `user_answers` 中 `q_1`（url_domain 问题的 ID）是否有值。Base URL 答案被完全忽略。

## 影响

- 预览页面所有 URL 都是相对路径（如 `/v2/pet/{{petId}}/uploadImage`）
- 保存到数据库后 URL 字段不完整
- 用户在 Step 3 填写的 Base URL 没有任何实际作用
- 生成的请求直接执行时会因为缺少 base URL 而失败

## 复现步骤

1. 上传 Petstore swagger.yaml
2. 在 Step 3 填写 Base URL 为 `https://petstore.swagger.io`
3. 生成后预览请求
4. URL 仍为 `/v2/pet/{{petId}}/uploadImage`，没有 `https://petstore.swagger.io` 前缀

## 预期行为

生成的 URL 应为 `https://petstore.swagger.io/v2/pet/{{petId}}/uploadImage`，或 base URL 应作为 `{{BASE_URL}}` 环境变量注入。

## 修复建议

在 `generate_requests()` 中检查 `user_answers` 中的 url_domain 值：

```python
base_url = user_answers.get("q_1", "").rstrip("/")
if base_url:
    url = base_url + url
```

同时考虑将 base URL 同时写入项目或集合的环境变量配置，以便测试执行时使用。
