## 描述

`_build_auth` 方法遍历所有 `user_answers` 的值，检查是否匹配 `"bearer"`、`"basic"`、`"apikey"`、`"none"` 关键词。如果用户在任意不相关的问题中填写了这些值（例如参数值输入了 `"none"`），认证方式会被静默覆盖。

## 根因

```python
for key, val in user_answers.items():
    if isinstance(val, str) and val in ("bearer", "basic", "apikey", "none"):
        auth_answer = val
        break
```

此循环扫描所有 `user_answers` 而非只读取认证类问题的答案。

## 复现步骤

1. 上传文档
2. 步骤 3 中在某个业务参数值输入框填写 "bearer" 或 "none"
3. 完成生成
4. 查看生成的请求认证方式被错误设置为 bearer 或 none

## 预期行为

应基于认证相关问题的 `question.id`（如 `q_auth`）读取答案，而非全量扫描：
```python
auth_answer = user_answers.get('auth')  # 或特定 question ID
```
