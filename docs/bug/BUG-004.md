## 描述

步骤 1 上传文档后，结果卡片始终显示以下错误信息：
- 文档格式：`JSON`（无论实际检测到的是 Swagger/OpenAPI/Postman/HAR）
- 接口数量：`0`（无论实际解析到多少个 endpoint）

后端实际返回的字段名是 `doc_type`（值为 `"swagger2"`、`"openapi3"`、`"postman"`、`"har"`），且不包含 `endpoint_count` 字段（端点数通过 `parsed_endpoints` 数组长度计算）。

## 根因

前端代码：
```js
detected_format: res.data.detected_format || res.data.format || 'JSON',
endpoint_count: res.data.endpoint_count || res.data.total_endpoints || 0
```

后端实际返回：
```json
{"id": 1, "doc_type": "swagger2", "parsed_endpoints": [...], ...}
```

## 复现步骤

1. 上传任意格式文档（Swagger/Postman/HAR）
2. 步骤 1 上传结果卡片
3. 格式标签显示"JSON"而非"Swagger 2.0"或"Postman"等
4. 接口数量显示"0"而非实际解析数

## 预期行为

- 格式标签正确显示文档类型（Swagger 2.0 / OpenAPI 3.0 / Postman / HAR）
- 接口数量正确显示解析到的 endpoint 总数

## 修复建议

前端：
```js
detected_format: res.data.doc_type || 'unknown',
endpoint_count: res.data.parsed_endpoints?.length || 0
```
