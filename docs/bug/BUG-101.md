## 描述

提示词配置页面的类型徽章使用三元表达式，只区分 `writer` 和 `reviewer`，其余所有类型（`vision`、`data_generator`、`field_classify`）都显示为"图片分析"。虽然 `vision` 类型恰巧显示正确，但 `data_generator` 和 `field_classify` 类型的提示词卡片上错误显示"图片分析"。

## 根因

```html
<span class="type-badge" :class="config.prompt_type">
  {{ config.prompt_type === 'writer' ? $t('promptConfig.writerPrompt') :
      (config.prompt_type === 'reviewer' ? $t('promptConfig.reviewerPrompt') :
      $t('promptConfig.imageAnalyzer')) }}
</span>
```

## 复现步骤

1. 进入提示词配置页面
2. 添加一个 `data_generator` 或 `field_classify` 类型的提示词
3. 保存后查看卡片上的类型徽章 — 显示"图片分析"

## 预期行为

`data_generator` 显示"测试数据生成提示词"，`field_classify` 显示"字段分类提示词"。
