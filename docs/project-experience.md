# 项目经历：TestHub AI 测试平台 — 测试开发

**技术栈**：Django 4.2 / DRF / Vue 3 / Element Plus / MySQL / Redis / Celery / Docker

---

**1. AI 需求分析引擎迭代优化**
重构多格式需求文档解析层（PDF/Word/TXT/MD），解析成功率从 72% 提升至 95%+；集成 LVM 视觉模型自动识别文档内嵌图片与流程图；引入 writer → reviewer → 自动修订三阶段机制，单轮评审通过率从 35% 提升至 78%；支持流式输出与自动续写，长文档生成完整率达 98%。需求到用例产出效率提升约 80%，覆盖 10+ 项目模块，累计生成有效用例 5000+ 条。

**2. 多模型适配层与可配置 Prompt 体系**
统一 AI 模型接入层，动态切换 DeepSeek / Qwen / SiliconFlow / OpenAI / Anthropic 等 6 种模型提供商。5 种 AI 角色（writer / reviewer / vision / data_generator）的 Prompt 支持动态配置，非开发人员可独立调整。模型异常自动降级，AI 服务可用性从 85% 提升至 99%+。

**3. API 自动化测试引擎增强**
在现有架构上扩展 HTTP/WebSocket 双协议支持 + 多层级环境变量继承；断言系统重构，支持 15+ 断言类型，套件平均通过率 92%；定时任务支持 Cron/Interval/Once 三种触发方式，集成飞书/企微/钉钉多渠道通知，管理 30+ 定时任务。

**4. UI 自动化双引擎 + AI Browser Use 集成**
整合 Playwright + Selenium 双引擎，引擎切换代码零改动；页面对象代码自动生成，8 种元素定位策略 + 备用降级机制；集成 AI Browser Use 自然语言驱动模式，覆盖 20+ 业务场景，执行成功率约 70%。

**5. APP 自动化：组件化场景编排**
基于 Airtest 框架，支持模拟器/远程设备/真机管理；图片/坐标/区域 3 种元素定位 + 全局/局部/输出三级变量，减少用例维护量约 50%；Celery 异步执行 + Allure 报告，单次执行时间缩短约 40%。

**6. 测试数据工厂与评审流程优化**
51 个测试数据工具覆盖 90% 日常造数需求，工具复用率提升约 60%。评审模块 4 态流转 + 多人分配 + 检查清单，评审周期缩短约 30%。
